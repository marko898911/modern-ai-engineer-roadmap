# How to Use This Roadmap

## The Rule

Do not try to finish every resource before building. Learn just enough to build
the next artifact, then use the artifact to reveal what you do not understand.

Every stage has five parts:

| Part | Meaning |
|---|---|
| Goal | Why the stage exists |
| Learn | Concepts that matter |
| Build | One artifact you can show |
| Measure | Numbers, tests, or checks |
| Exit criteria | What proves you can move on |

## Pace

This roadmap can be walked at different speeds.

| Pace | Time | Use it when |
|---|---:|---|
| Fast survey | 3 to 4 months | You already code and want the big map |
| Student path | 9 to 15 months | You are learning seriously beside school |
| Deep mastery | 18 to 30 months | You want strong portfolio and systems depth |

The mastery path is long because modern AI engineering is not one job. It
touches software engineering, ML, product systems, distributed infrastructure,
security, and sometimes hardware.

## Weekly Study Loop

Use this loop for every week.

1. Pick one concept from the current stage.
2. Read or watch one primary learning resource.
3. Implement the smallest working version.
4. Measure one thing: accuracy, latency, recall, cost, memory, throughput, or error rate.
5. Write a short note: what worked, what failed, what changed.

## When to Skip

Skip a stage only if you can pass its exit criteria without help. Do not skip
because a topic feels boring. Boring foundations become expensive bugs later.

Good skips:

- You already build Python packages and can test/debug them.
- You already trained models and can explain overfitting, leakage, and evaluation.
- You already shipped a model service and monitored latency and errors.

Bad skips:

- You can call an API but cannot evaluate the output.
- You can use an agent framework but cannot write a simple agent loop.
- You can run a model but cannot explain memory, batching, or latency.

## The First 30 Days

If you are a student starting now:

| Week | Focus | Output |
|---|---|---|
| 1 | CLI, Git, Python, notebooks | public learning log |
| 2 | Python data stack | small data analysis repo |
| 3 | ML basics | train/evaluate a classifier |
| 4 | LLM basics | tokenizer and prompt notebook |

After 30 days, you should have four small artifacts and enough confidence to
keep walking.

## How This Fixes the Old Roadmap Problem

The old AI hardware roadmap has impressive content, but it asks a beginner to
hold too many tracks at once: digital design, embedded systems, AI workloads,
Jetson, FPGA, compilers, agents, and chip design. This roadmap uses a different
shape:

- one main path first
- specialist tracks later
- every stage has a student-friendly artifact
- advanced hardware and ZKML are introduced after core AI engineering
- pages are shallow and linked instead of deeply nested

You should always know the next move.
