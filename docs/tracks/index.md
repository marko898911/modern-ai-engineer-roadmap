# Role Tracks

The stages are the main path. Tracks are shortcuts through the same material
for a specific goal. Use a track after you understand the full map.

## AI Application Engineer

Builds useful AI products with strong evaluation and product judgment.

| Priority | Stages |
|---|---|
| Required | 0, 1, 2, 4, 5, 7 |
| Important | 6, 9 |
| Optional depth | 3, 8 |

Portfolio:

- evaluated RAG app
- prompt and eval registry
- user feedback loop
- deployment with monitoring

## Choosing a Track

Choose based on what you enjoy debugging:

| If you enjoy debugging... | Choose |
|---|---|
| user behavior and product quality | AI Application Engineer |
| model behavior and evaluation | LLM Engineer |
| loops, tools, memory, and workflows | Agent Engineer |
| deployment and reliability | ML Systems Engineer |
| latency, memory, and GPUs | Inference and Acceleration Engineer |
| attacks, trust, proofs, and contracts | AI Security, Blockchain, and ZKML Engineer |
| papers and experiments | Research Engineer |
