# Stage 0: Orientation

<span class="stage-badge">0</span> Learn what modern AI engineering is before you choose tools.

## Goal

Build a map in your head:

- AI as a field: ML, deep learning, foundation models, agents
- AI engineering as a job: building useful systems around models
- model choices: API model, open-weight model, fine-tuned model, local model
- system choices: prompt, RAG, agent, workflow automation, training, serving
- trust choices: evaluation, guardrails, privacy, monitoring, security

## Learn

| Topic | What to understand |
|---|---|
| AI vs ML vs deep learning | AI is the goal, ML is one method, deep learning is one ML family |
| Foundation models | general models adapted with prompting, context, tools, or fine-tuning |
| AI engineering stack | product, prompts, context, model, serving, data, evals, monitoring |
| Model tradeoffs | quality, latency, price, privacy, context length, licensing |
| Learning workflow | notes, projects, measurements, public portfolio |

## Build

Create a `learning-log` repo with:

- `README.md` describing your goal and weekly schedule
- `notes/` for short concept notes
- `projects/` for each artifact in this roadmap
- `decisions.md` for model/tool choices and why you made them

## Measure

Track:

- hours studied
- concepts learned
- project status
- blockers
- useful links

## Exit Criteria

Move on when you can explain:

- the difference between ML engineering and AI engineering
- what an LLM application adds around a model
- why evaluation is required before fine-tuning
- why an agent needs tools, state, and safety boundaries
