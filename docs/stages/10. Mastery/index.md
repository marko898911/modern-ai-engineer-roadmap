# Stage 10: Mastery

<span class="stage-badge">10</span> Choose a specialization and build a capstone that proves end-to-end judgment.

## Goal

Mastery means you can own a real AI system: design it, build it, evaluate it,
secure it, deploy it, optimize it, and explain the tradeoffs.

## Choose a Capstone

| Capstone | What it proves |
|---|---|
| Production RAG platform | AI app design, evals, observability, feedback |
| Agent workflow platform | tools, memory, planning, tracing, security |
| LLM serving platform | infrastructure, batching, optimization, cost control |
| Edge AI assistant | local inference, hardware constraints, privacy |
| AI security lab | threat modeling, red teaming, mitigations |
| ZKML verifier demo | cryptographic verification and model constraints |
| Research reproduction | paper reading, implementation, measurement |

## Required Capstone Sections

Your final portfolio project should include:

- problem statement
- architecture diagram
- model choice and alternatives
- data and evaluation plan
- implementation
- deployment instructions
- latency, cost, and quality metrics
- security and privacy analysis
- failure cases
- future work

## Mastery Criteria

You are operating at mastery level when you can:

- design before coding
- choose simpler systems when they solve the problem
- create evaluation data before trusting model behavior
- debug across prompt, retrieval, model, serving, and product layers
- explain cost and latency tradeoffs
- secure model access, tools, data, and logs
- teach the system to another engineer
