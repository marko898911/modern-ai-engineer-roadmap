# Use Examples in Your Prompt

## Purpose

Show how examples teach the model the pattern you expect.

## Contributor Scope

Cover:

- normal examples
- edge cases
- refusal examples
- tool-use examples
- bad examples to avoid

## Build

Add one good example and one edge-case example to a prompt, then test whether
the output becomes more stable.

## Exit Criteria

You can use examples without making the prompt too long or too narrow.
